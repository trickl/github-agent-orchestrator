# Gap Analysis Issue Template

## Gap Analysis (Plan â†’ Next Development Step)

### Purpose
Identify **one** logical next development item by comparing the long-term goal to current capabilities.

### Title
**Identify the next most important development gap**

### Context
You are asked to identify a single, concrete next development task.

### Inputs
- `/planning/vision/goal.md`
- `/planning/state/system_capabilities.md`
- Any other relevant planning documents

### Task
1. Read and understand the long-term goal.
2. Read and understand the current system capabilities.
3. Identify the most important gap between the two.
4. Propose **ONE** development task that would meaningfully reduce that gap.

### Constraints
- Do not modify any code.
- Do not create GitHub issues.
- Do not propose multiple tasks.
- Do not solve the task.

### Output
- Create a single new file in `/planning/issue_queue/pending/`.
- The file should describe the proposed development task clearly and concisely.
- The **first line** of the file must be a friendly task name.

### Tone
Analytical, pragmatic, and aligned with the stated mission.
